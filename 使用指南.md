## 基础操作

### 1️⃣ 添加任务

1. 在顶部输入框填写任务标题
2. （可选）选择截止日期
3. 选择分类：💼工作 / 🏠生活 / 📚学习 / 📌其他
4. 选择优先级：🔴高 / 🟡中 / 🟢低
5. 选择颜色标签（6 种可选）
6. 点击 ➕ 添加任务

### 2️⃣ 管理任务

| 操作 | 方法 |
|------|------|
| 完成任务 | 点击左侧圆形复选框 |
| 编辑任务 | 点击 ✏️ 按钮 |
| 删除任务 | 点击 🗑️ 按钮 |
| 调整顺序 | 按住拖拽 |

### 3️⃣ 子任务

点击任务下方的 **+ 添加子任务** 按钮，可以把大任务拆分成多个小步骤。每个子任务可以独立勾选完成。

## 高级功能

### 🔍 搜索与筛选

- **搜索框**：实时匹配任务标题
- **状态筛选**：全部 / 待完成 / 已完成 / 已过期
- **分类筛选**：按工作/生活/学习/其他
- **优先级筛选**：按高/中/低

可以组合使用多个筛选条件。

### ✅ 批量操作

1. 在任务行勾选 **"选中"** 复选框
2. 在底部点击批量操作按钮：
   - ✅ 批量完成
   - 📁 批量移动到指定分类
   - 🗑️ 批量删除

### 📤📥 导入/导出

- **导出**：点击 📤 导出，会下载一个 JSON 文件
- **导入**：点击 📥 导入，选择之前导出的 JSON 文件
- 文件名格式：`tasks-2026-06-25.json`

### 📊 周报统计

底部柱状图显示本周（周日 → 周六）每天的完成任务数量。当天高亮显示。

### 🌓 深色模式

点击右上角 🌓 按钮切换深色/浅色主题。设置会自动保存。

## GitHub 云同步

### 🔑 创建 Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token** → **Generate new token (classic)**
3. 填写 Note（例如 `todo-app`）
4. **勾选 `repo` 权限**
5. **Expiration** 建议选 90 天（定期轮换）
6. 点击 **Generate token**
7. **复制并保存** Token（页面关闭后无法再次查看）

### ⚙️ 配置同步

1. 点击应用右上角 **⚙️** 按钮
2. 填写以下信息：
   - **GitHub 用户名**：你的 GitHub 用户名（如 `amainoyo`）
   - **📦 数据仓库名**（建议私有）：例如 `todo-data`
   - **分支名**：默认 `main`
   - **文件路径**：默认 `data/tasks.json`
   - **Personal Access Token**：你刚才创建的 Token
3. 点击 **+ 创建** 按钮 → 自动调用 GitHub API 创建私有仓库
4. **勾选** "启用 GitHub 同步"
5. 点击 **保存**

### 🏗️ 推荐架构（数据分离）

```
your-username/todo-app (公开仓库)        ← 应用代码
├── index.html
└── ...

your-username/todo-data (私有仓库)       ← 你的任务数据  
└── data/
    └── tasks.json
```

**优势**：
- ✅ 应用代码公开（GitHub Pages 可用）
- ✅ 任务数据私有（只有你能访问）
- ✅ 一键创建私有仓库

### 📡 同步机制

- **首次加载**：从 GitHub 拉取已保存的任务
- **每次修改**：1.5 秒后自动推送到 GitHub
- **降级保护**：GitHub 失败时保留本地存储
- **冲突解决**：最后写入获胜（适合单用户多设备）

### 🔄 多设备使用

只要在不同设备上用同一个 Token 配置，就能自动同步所有任务。

### 👥 多用户独立使用

站点是公开的，任何人都可以：
1. 访问 https://amainoyo.github.io/todo-app/
2. 用自己的 GitHub 账号配置
3. 数据存储到**自己的私有仓库**
4. 互不干扰

## 番茄钟使用

### 🍅 启动

1. 点击 **▶ 开始** 按钮
2. 计时器从 25:00 开始倒计时
3. 25 分钟后自动结束，发出提示音
4. 自动切换到 5 分钟休息模式
5. 休息结束后再次提示
6. 每完成 4 个番茄，底部小圆点会记录

### 操作

| 按钮 | 功能 |
|------|------|
| ▶ 开始 | 启动计时 |
| ⏸ 暂停 | 暂停计时（按钮文字变为暂停时） |
| 🔄 重置 | 重置到 25:00 |

## 常见问题

### Q: 任务数据存在哪里？
**A**: 默认存储在浏览器 LocalStorage 中。启用 GitHub 同步后会保存到 GitHub 仓库。

### Q: 换浏览器/电脑能用吗？
**A**: 
- 未启用 GitHub 同步：不能（数据在本地）
- 启用 GitHub 同步：可以，用同一个 Token 配置即可

### Q: Token 安全吗？
**A**: Token 仅保存在你的浏览器 LocalStorage 中，不会上传到任何服务器。但请勿在公共电脑使用，避免泄露。

### Q: GitHub 同步失败怎么办？
**A**: 右上角会显示红色错误提示。检查：
1. Token 是否有效（未过期、未撤销）
2. 网络是否畅通
3. 仓库是否存在且有写入权限

### Q: 移动端能用吗？
**A**: 支持！响应式设计已优化手机屏幕。横屏和竖屏都能正常操作。

### Q: 数据会冲突吗？
**A**: 简单实现是"最后写入获胜"。如需多人协同编辑，建议用更专业的工具。

### Q: 同步到 GitHub 后别人能看到吗？
**A**: 取决于仓库可见性：
- **公开仓库**：任何人都能在 https://github.com/你的用户名/你的仓库/blob/main/data/tasks.json 看到明文任务
- **私有仓库**：只有你能访问

**强烈建议**：使用应用内的 **+ 创建** 按钮自动创建私有数据仓库（如 `todo-data`），与公开的应用仓库（`todo-app`）分离。

### Q: 站点是公开的，别人也能用吗？
**A**: 是的！任何人都可以：
1. 访问 https://amainoyo.github.io/todo-app/
2. 用自己的 GitHub 账号 + Token 配置
3. 数据存储到**自己的私有仓库**
4. 互不干扰

这是这个 app 的核心优势——**一次部署，所有人可用，各自私有**。
